# 🚀 실행 명령어 가이드

## 📋 빠른 실행 가이드

### 1️⃣ 백엔드 실행 (먼저 실행)

```bash
# 백엔드 디렉토리로 이동
cd smart-sejong-backend

# Gradle로 실행
./gradlew bootRun
```

**Windows PowerShell:**
```powershell
cd smart-sejong-backend
.\gradlew.bat bootRun
```

**Windows CMD:**
```cmd
cd smart-sejong-backend
gradlew.bat bootRun
```

✅ 백엔드가 `http://localhost:8080`에서 실행됩니다.

---

### 2️⃣ 프론트엔드 실행 (백엔드 실행 후)

**새 터미널 창을 열고:**

```bash
# 프론트엔드 디렉토리로 이동
cd smart-sejong-frontend

# 의존성 설치 (최초 1회만)
npm install

# 개발 서버 실행
npm run dev
```

**Windows PowerShell:**
```powershell
cd smart-sejong-frontend
npm install
npm run dev
```

✅ 프론트엔드가 `http://localhost:3000`에서 실행됩니다.

---

## 📝 전체 실행 순서

### 터미널 1 (백엔드)
```bash
cd smart-sejong-backend
./gradlew bootRun
```

### 터미널 2 (프론트엔드)
```bash
cd smart-sejong-frontend
npm run dev
```

---

## 🌐 접속 주소

- **프론트엔드**: http://localhost:3000
- **백엔드 API**: http://localhost:8080
- **Swagger UI**: http://localhost:8080/swagger-ui.html

---

## ⚠️ 주의사항

1. **백엔드를 먼저 실행**해야 프론트엔드가 정상 작동합니다
2. **두 개의 터미널**이 필요합니다 (백엔드 1개, 프론트엔드 1개)
3. 백엔드가 실행 중이 아니면 카카오 로그인 등 API 호출이 실패합니다

---

## 🛑 종료 방법

- **백엔드**: 터미널에서 `Ctrl + C`
- **프론트엔드**: 터미널에서 `Ctrl + C`

---

## 🔧 문제 해결

### 백엔드 실행 오류
```bash
# Gradle 캐시 정리 후 재시도
cd smart-sejong-backend
./gradlew clean
./gradlew bootRun
```

### 프론트엔드 실행 오류
```bash
# node_modules 삭제 후 재설치
cd smart-sejong-frontend
rm -rf node_modules package-lock.json
npm install
npm run dev
```

**Windows:**
```powershell
cd smart-sejong-frontend
Remove-Item -Recurse -Force node_modules, package-lock.json
npm install
npm run dev
```
