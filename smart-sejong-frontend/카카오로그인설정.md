# 카카오 로그인 설정 가이드

## ✅ 구현 완료

카카오 로그인이 정상적으로 작동하도록 구현되었습니다.

## 🔄 로그인 흐름

1. **프론트엔드**: 사용자가 "카카오로 시작하기" 버튼 클릭
2. **프론트엔드**: 백엔드에서 카카오 로그인 URL 조회 (`/api/auth/kakao/login-url`)
3. **사용자**: 카카오 로그인 페이지에서 로그인
4. **카카오**: 백엔드 콜백 URL로 인가 코드 전달 (`/login/oauth2/code/kakao`)
5. **백엔드**: 인가 코드로 JWT 토큰 발급 후 HTML 페이지 반환
6. **HTML 페이지**: localStorage에 토큰 저장 후 프론트엔드로 리다이렉트
7. **프론트엔드**: localStorage에서 사용자 정보 로드하여 인증 상태 업데이트

## ⚙️ 카카오 개발자 콘솔 설정

카카오 개발자 콘솔(https://developers.kakao.com)에서 다음 설정이 필요합니다:

### Redirect URI 등록
```
http://localhost:8080/login/oauth2/code/kakao
```

**중요**: 
- 백엔드가 실행되는 포트(기본: 8080)로 설정해야 합니다
- 프론트엔드 포트(3000)가 아닙니다!

### 현재 설정 확인
백엔드 `application.properties` 파일에서 확인:
```properties
spring.security.oauth2.client.registration.kakao.redirect-uri=http://localhost:8080/login/oauth2/code/kakao
```

## 🐛 문제 해결

### 카카오 로그인이 안 되는 경우

1. **카카오 개발자 콘솔 확인**
   - Redirect URI가 `http://localhost:8080/login/oauth2/code/kakao`로 등록되어 있는지 확인
   - Client ID와 Client Secret이 올바른지 확인

2. **백엔드 서버 확인**
   - 백엔드가 `http://localhost:8080`에서 실행 중인지 확인
   - 콘솔에서 에러 메시지 확인

3. **프론트엔드 확인**
   - 브라우저 콘솔(F12)에서 에러 확인
   - 네트워크 탭에서 API 요청 상태 확인

4. **CORS 설정 확인**
   - 백엔드 `SecurityConfig.java`에서 `http://localhost:3000`이 허용되어 있는지 확인

## 📝 참고사항

- 카카오 로그인 후 백엔드 콜백 페이지에서 자동으로 프론트엔드로 리다이렉트됩니다
- JWT 토큰은 localStorage에 저장됩니다
- 사용자 정보도 localStorage에 저장되어 페이지 새로고침 시에도 유지됩니다
