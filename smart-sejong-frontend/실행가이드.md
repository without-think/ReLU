# Smart Sejong Frontend 실행 가이드

## ✅ 검토 완료 사항

- ✅ 모든 의존성 패키지 확인
- ✅ TypeScript 설정 확인
- ✅ 경로 별칭(@) 설정 확인
- ✅ 모든 import 문 확인
- ✅ 컴포넌트 구조 확인
- ✅ 린터 오류 없음

## 🚀 실행 방법

### 1. 필수 요구사항
- **Node.js 18 이상** 설치 필요
- **npm** 또는 **yarn** 패키지 매니저

### 2. 설치 및 실행

```bash
# 1. 프론트엔드 디렉토리로 이동
cd smart-sejong-frontend

# 2. 의존성 패키지 설치 (최초 1회만)
npm install

# 3. 개발 서버 실행
npm run dev
```

### 3. 접속
브라우저에서 자동으로 열리거나, 수동으로 접속:
```
http://localhost:3000
```

## ⚙️ 백엔드 연동

### 백엔드 서버 실행
백엔드가 `http://localhost:8080`에서 실행 중이어야 합니다.

```bash
# 백엔드 디렉토리에서
cd smart-sejong-backend
./gradlew bootRun
# 또는
gradlew.bat bootRun  # Windows
```

### API 프록시 설정
Vite 개발 서버가 자동으로 `/api/*` 요청을 백엔드로 프록시합니다.
- 프론트엔드: `http://localhost:3000`
- 백엔드: `http://localhost:8080`
- 프록시 설정: `vite.config.ts`에 이미 구성됨

## 📝 환경 변수 (선택사항)

백엔드가 다른 주소에서 실행되는 경우 `.env` 파일 생성:

```env
VITE_API_BASE_URL=http://localhost:8080
```

## 🔧 빌드 및 배포

```bash
# 프로덕션 빌드
npm run build

# 빌드 결과물 미리보기
npm run preview
```

빌드 결과물은 `dist/` 폴더에 생성됩니다.

## 🐛 문제 해결

### 포트가 이미 사용 중인 경우
```bash
# vite.config.ts에서 포트 변경
server: {
  port: 3001,  # 다른 포트로 변경
}
```

### 의존성 설치 오류
```bash
# node_modules 삭제 후 재설치
rm -rf node_modules package-lock.json
npm install
```

### TypeScript 오류
```bash
# 타입 체크만 실행
npx tsc --noEmit
```

### 백엔드 연결 오류
- 백엔드 서버가 실행 중인지 확인
- CORS 설정 확인 (SecurityConfig.java)
- API 엔드포인트 확인

## 🎯 주요 기능 테스트

1. **로그인 페이지**: `/login`
   - 포털 인증 테스트

2. **학습 현황**: `/learning`
   - CSV 업로드 테스트
   - 대시보드 확인

3. **AI 추천**: `/recommendation`
   - 필터 설정 테스트
   - 추천 생성 테스트

4. **내 시간표**: `/timetable`
   - 시간표 생성/저장 테스트

5. **그룹 협동**: `/group`
   - 그룹 생성/참가 테스트

## ⚠️ 주의사항

1. **인증 토큰**: 로그인 후 토큰이 localStorage에 저장됩니다.
2. **백엔드 연동**: 모든 API 호출은 백엔드가 실행 중이어야 작동합니다.
3. **카카오 로그인**: 실제 카카오 SDK 연동이 필요합니다 (현재는 UI만 구현).
